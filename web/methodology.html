Organisation du développement

La DSI adopte une méthodologie basée sur Git afin d'assurer la qualité et la traçabilité du code.

Stratégie de branches

  • main : version stable en production
  • develop : intégration des fonctionnalités
  • feature/* : développement des nouvelles fonctionnalités
  • fix/* : corrections de bugs

Processus de contribution

  1. Créer une branche à partir de develop
  2. Développer la fonctionnalité ou corriger le bug
  3. Committer les modifications
  4. Ouvrir une Pull Request
  5. Attendre validation avant fusion

Bonnes pratiques

  • Ne jamais pousser directement sur main
  • Rédiger des messages de commit explicites
  • Tester les modifications avant soumission
  • Documenter les changements importants
